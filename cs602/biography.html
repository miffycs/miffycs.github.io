Biography

My name is Miffy Chen, I'm a part-time master's student in BU's Computer Science program in the MET college. I also went to BU for my undergraduate degree, which was in Psychological and Brain Sciences. I enjoyed learning about psychology as a discipline and academic subject, however, I didn't really like the jobs available for someone who only has a bachelor degree (professionals usually pursue a Psy.D or PHD). After working for a few years, I decided I want to pursue a degree in CS, specifically to join the gaming industry as a game developer. However, the more I learn about CS and all the possible career paths within it, the less sure I am of what I really want to do after I graduate. I will be graduating in May 2020. Currently I'm working towards becoming a full-stack developer, with a slight focus on front-end, because I like to build web apps with beautiful UI/UX.

I started attending local hackathons to make up for my lack of working experiences as a developer, which sparked my interest in web development, and helped beef-up my resume for applying to internships. Early last year, I started my first internship with local start-up named BrainCo, they make EEG headbands that allow for real-time brainwave visualization, which they use to help students and athletes train for focus as well as relaxation. While working here, I picked up web dev using React.js and Node.js to build cross-platform software rendered with Electron.js. Around June, I left BrainCo to start a summer internship with AWS. I had a great time and learned a ton on cloud computing and building web-apps using a hybrid-cloud or solely cloud-based structure, and am planning on taking their exams to become an AWS Certified Cloud Practitioner / Solutions Architect Associate. I am taking this class because much of my web dev knowledge is self-taught, I google problems when I encounter them, so I don't have a solid foundation of web dev. I'm hoping to walk away from this course feeling confident that I understand how things work together, instead of just how to build them and make them work. Some topics I hope we will touch on include: employing MVC (model-view-controller) structures on large scale applications, connecting relational (sequel) and non-relational (no-sequel) databases to web apps, and traditional back-end structures (when not using a cloud service platform). I'm currently working full-time as a Cloud Software Engineer with a company based in Waltham, so I really appreciate the flexibility of taking an online course, and being able to view lectures on my own time.
